Forecast-Based Signal Audit

Examining how issuance decisions align with published HKO criteria and expected persistence

This section analyzes the forecast-side rationale for issuing Signal No. 8 based on expected lowland wind coverage and persistence. Unlike the observation tiers (Verified / Pattern-Validated / Unverified), the audit examines whether the decision context matched published criteria at the time of issuance.

Important: This is research transparency only. HKO applies expert judgement that integrates more data sources than our simplified model (e.g., mesoscale analysis, pressure gradients, offshore station trends).

Model Inputs

Algorithm Logic

The audit script estimates whether the minimum station count and persistence criteria would have been reasonable based on shortly pre-issuance intervals. It flags: (a) lead-time gap, (b) initial coverage sufficiency, (c) persistence plausibility.

Code reference: hko_compliant_algorithm.py & run_hko_audit.py
